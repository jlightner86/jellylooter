üìÅ Folder Naming Formats

üè∑Ô∏è

Why Use Provider IDs?

Including IMDB/TVDB/TMDB IDs in folder names helps Jellyfin, Plex, Sonarr, and Radarr correctly identify your media - especially for anime, foreign films, or titles with common names.

üìã

Available Formats

Choose from 6 naming formats in Settings ‚Üí Advanced Settings:
Standard - Movie (2010) / Show Name
IDs (Space) - Movie (2010) tt1234567 12345
IDs (Braces) - Movie (2010) {imdb-tt1234567} {tmdb-12345}
IDs (Brackets) - Movie (2010) [imdb-tt1234567]
TMDB Only - Movie (2010) {tmdb-12345}
IMDB Only - Movie (2010) tt1234567

üëÄ

Live Preview

The settings page shows a live preview of how your folders will be named for both movies and TV shows as you select different formats.

üîç Metadata API Integration

üîë

API Keys

Get free API keys from these providers to enable metadata lookup:
TMDB - Best for movies & TV, includes IMDB IDs
TVDB - Best for TV shows & anime
OMDb - IMDB data fallback (1,000/day limit)

‚öôÔ∏è

Lookup Timing

Choose when metadata is fetched in Settings ‚Üí Advanced Settings:
On Browse - Slower browsing, but IDs shown before download
On Download - Faster browsing, IDs fetched at download time

üíé

Metadata Caching (Pro)

Pro users get automatic metadata caching - once an item is looked up, the result is stored locally for instant access on repeat browsing.

üîÑ

Fallback Chain

JellyLooter first checks the remote server for IDs. If missing, it queries TMDB ‚Üí TVDB ‚Üí OMDb until it finds what it needs.

Auto-Retry Failed Downloads

Enable in Advanced Settings to automatically retry downloads that fail due to connection errors, timeouts, or server issues. Configure:
Retry attempts - How many times to retry (1-10, default 3)

Download Limit Override (Pro)

Pro users can enable "Override download limit" to allow more than 10 concurrent downloads (up to 50). ‚ö†Ô∏è Warning: High concurrent downloads can overload remote servers, cause rate limiting or bans, and affect other users on shared servers. Use responsibly.

Connection Settings

Connection timeout - How long to wait for server response (default 30s)
Chunk size - Download buffer size in KB (default 64KB, higher = faster but more memory)

üéõÔ∏è

Transcode Presets

Original - No transcoding, keep source file
H.264 - Compatible with most devices (.mp4)

‚ö°

Hardware Acceleration

JellyLooter supports NVIDIA NVENC, Intel QuickSync, and AMD VAAPI. For NVENC, use Dockerfile.nvidia and pass --gpus all to Docker. The app auto-detects available encoders and falls back to software if needed.

üîÄ

Separate Workers

Downloads and transcodes run in parallel with separate worker pools. Configure 1-10 download workers and 1-5 transcode workers. This means downloads continue while files are being transcoded.

üíæ

Cache Drive

Enable "Use cache drive" to transcode files to a fast SSD first, then move them to the final destination. This reduces wear on slower drives, improves performance with network storage (NAS/SMB/NFS), and keeps partial files separate from your media library.

üóëÔ∏è

Clear Cache

Use the "Clear Cache" button to remove partial/failed transcode files from the cache drive. The cache status shows how many files and total size are currently stored.

‚öñÔ∏è

Skip if Larger

Enable "Skip if larger" to automatically keep the original file when transcoding would increase file size. This prevents wasted space from inefficient transcodes (e.g., highly compressed originals).

üìä

Transcode Statistics

Track your transcoding efficiency: files processed, files skipped (larger), total space saved, and average reduction percentage. Stats persist across restarts and can be reset anytime.

üö¶

Queue Limit

The transcode queue is limited to 10 pending files to prevent memory issues. Downloads will pause if the queue is full and resume automatically when space is available.

üîÑ Download Resume (Pro)

üíé

Pro Feature

Download resume is available for Pro license holders. Interrupted downloads can be resumed from where they left off.

üìÅ

How It Works

Downloads in progress are saved as .partial files. If JellyLooter restarts or a download is interrupted, it will automatically resume from the last saved position.

‚è±Ô∏è

Auto-Save

Progress is saved every 30 seconds during download. After completion, the .partial extension is removed and the file is renamed to its final name.

‚ùì Frequently Asked Questions

Q: What's the difference between Ko-fi donations and Pro license?
Ko-fi donations are just tips to support development - you get nothing except our gratitude. Pro licenses ($10 lifetime) unlock all Pro features: notifications, GPU transcoding, download scheduling, *arr integration, custom themes, and remove the banner ad.
Q: Where do ratings come from?
Ratings are pulled directly from your Jellyfin/Emby server's metadata. JellyLooter doesn't make external API calls for ratings. If your server has IMDB/TMDB ratings in its database (from library scans), they'll appear on posters.
Q: Why don't I see 4K/HDR badges on my content?
Quality badges require MediaStreams data from Jellyfin/Emby. Run a full library scan on your remote server to populate this data. Also ensure "Show quality badges" is enabled in Advanced Settings.
Q: Can I use JellyLooter with Plex?
No, JellyLooter only supports Jellyfin and Emby servers. Plex uses a different API that is not compatible.
Q: Is my API key/password stored securely?
API keys are stored in your local config file. Passwords are hashed with bcrypt before storage. The config file is only accessible within your Docker container's /config volume.
Q: How does duplicate detection work?
JellyLooter scans your local Jellyfin/Emby server and builds a cache of IMDB/TMDB IDs. When browsing remote content, items that exist locally are highlighted with a green border. Click "Rebuild Cache" to refresh.
Q: Can I download entire seasons/series at once?
Yes! Navigate to a series or season and select it, then click Download. JellyLooter will queue all episodes. You can also Ctrl+click to select multiple items.
Q: What happens if a download fails?
Failed downloads are logged in the Activity Log. Partial files are automatically cleaned up. You can retry by selecting the item and downloading again.
Q: How do I get GPU transcoding working on Unraid?
For NVIDIA GPUs, add --runtime=nvidia to Extra Parameters and set NVIDIA_VISIBLE_DEVICES=all (or your GPU UUID). For Intel QuickSync, pass through /dev/dri. Transcoding is a Pro feature.
Q: My license key isn't working. What do I do?
Make sure you're entering the key exactly as received (with dashes). Keys are validated with Gumroad's API. If you're offline, the key will be trusted based on format. Contact us on GitHub if issues persist.
